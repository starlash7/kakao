# CLAUDE.md

이 저장소에서 작업하는 에이전트는 아래 규칙을 따른다.

## 1. 빌드/린트

- `build`/`lint`는 ait 파일 생성 및 출시 직전에만 실행한다.

## 2. 코드 수정 원칙

- 불필요한 문구, 중복 멘트를 금지한다.
- 요청 없이 기존 멘트/문구를 임의 수정하지 않는다.
- 요청 범위 밖 파일을 수정하지 않는다.
- 기존 기능을 삭제하지 않는다.
- 새 의존성을 추가하기 전에 stdlib/기존 패키지로 가능한지 먼저 확인한다.
- 최소 코드 우선으로 작업한다.

## 3. 코딩 가이드

- Karpathy-guidelines를 기본 규칙으로 한다.
- 파일 하나는 역할 하나만 가진다.
- 파일이 200줄을 넘으면 분리를 검토한다.
- utils 잡탕 파일을 만들지 않는다.
- 함수명은 동사+명사로 명확하게 작성한다.
  - 예: `get_station_list`, `calculate_reward`
